--****TOP 5: Meilleurs Papiers sur la Blockchain dans la Santé


La Blockchain est la nouvelle technologie dont tout le monde parle. Cependant, beaucoup ne comprennent pas ce que cest, comment cela fonctionne et comment elle peut être utilisée. 
La Blockchain et sa première application, le Bitcoin, sont devenus très populaires dans le secteur financier, avec de nombreuses banques internationales qui investissent dans cette technologie. 
Mais ses applications ne se limitent pas à la finance et à la banque.
Un autre secteur qui pourrait bénéficier de la Blockchain est la santé.

Le système de santé doit évoluer et innover. 
Malgré de grands progrès dans lapproche thérapeutique avec de nouveaux médicaments en développement tous les ans,
celui-ci manque de fluidité, de modernité et defficacité.


Aujourdhui, il est encore difficile dassurer la traçabilité complète des medicaments. 
Les médecins commettent des erreurs médicales parce quils nont pas accès à temps au dossier médical complet dun patient. 
Les données des essais cliniques sont difficiles à obtenir et à partager. La Blockchain pourrait être le levier de changement dans le système de santé.

******************************* multi-Orbac
Multi-OrBAC est une extension du OrBAC pour les systèmes multi organisationnels
complexes, hétérogènes, interopérables et distribués. Cest un modèle de sécurité dynamique
et adaptable, permettant dun côté de spécifier des politiques de sécurité différentes dans
chaque organisation, et de lautre dimposer des règles pour les interactions entre organisations
qui soient compatibles avec les politiques de chaque organisation. Ce modèle vise donc des
applications et systèmes complexes, hétérogènes, interopérables et distribués.
****Multi-OrBAC garde la méthodologie très pertinente dOrBAC
****Le principal défaut de Multi-OrBAC réside dans le fait que la définition de la politique
de sécurité de chaque organisation doit prendre en compte, donc connaître, 
des entités appartenant à dautres organisations, et suppose donc une confiance
entre les organisations pour ce qui concerne la gestion de ces entités.

*******************************VO(Virtual Organization)
****Une Organisation Virtuelle(OV) est un ensemble dorganisations reliées par les technologies
 dinformation (TI), poursuivant la réalisation commune dun projet ou dune activité économique.
***Une Organisation Virtuelle définit une communauté virtuelle formée de plusieurs organisations
 ou entreprises qui unissent leurs compétences et ressources pour répondre à une opportunité
 quelles nauraient pu prendre en charge seules.
****Cette alliance est en général définie pour une durée limitée : une fois le bien ou service livré,
le regroupement est démantelé. Les infrastructures des technologies de linformation sont au coeur
de lorganisation virtuelle et le support de ses activités

## Allows people in group wheel to run all commands
%wheel ALL=(ALL) ALL

Le groupe wheel a les droits sudo. Vous allez donc ajouter lutilisateur user-ansible dans le  groupe wheel sur tous les nodes :
ansible -i inventaire.ini -m user -a 'name=user-ansible groups=wheel append=yes ' --user root --ask-pass all

Un moyen de vérifier que user-ansible a bien les droits sudo est de relancer la commande précédente, mais en ajoutant de nouvelles options :

ansible -i inventaire.ini -m user -a 'name=user-ansible groups=wheel append=yes ' --user user-ansible --ask-pass --become --ask-become-pass all

Vous allez utiliser le module authorized_key pour enregistrer la clé publique de l'utilisateur user-ansible sur tous les nodes. Ainsi, vous pourrez vous connecter aux nodes sans saisir de mot de passe SSH.

ansible -i inventaire.ini -m authorized_key -a 'user=user-ansible state=present key="{{ lookup("file", "/home/user-ansible/.ssh/id_ecdsa.pub") }}"' --user user-ansible --ask-pass --become --ask-become-pass all

des motifs simples ou complexes respectivement, dans un fichiers.

on distingue deux type d'expression regulier. les expression reguliere simple
et les expression reguliere etandu

les expression reguliere de base sont les suivant:
[],*,^$
pour faire un recherche un mot ou une ligne dans un fichier on faire appelle
aux expression regulier de base
utilisation des ces expressions reguliere
le caractere . (correspond a n'importe qu'elle caractere)

-- le crochet [] correspondent a un seul caractere de liste ou

-- chaque caractere possible peut etre lister [abcd] ou fourni
sous la forme d'une plage [a-d], condition que la plage soit
dans le bon ordre.


pour affichier la ligne qui contient les caractere entre '[0-9]' il faut exercute:
grep '[0-9]' filename
em ajoutant ^ ceci nous permet de faire une negation
le caratere circonflexe ^ est utiliser pour s'assurer qu'un motif apparait au bebut de la ligne
le deuxieme caractere $ peut etre utilise pour s'assurer qu'un motif apparait a la fin de ligne

ex: grep '^root' filename //cette expression nous donne tous les ligne qui commence par le mot root.
ex2: grep 'root$' filename // cette expression nous donne tous les lignes qui se termine par root.

autre expression regulier de base c'est \.
la commande grep 'ro\*' filename recherche le mots ro* dans le fichier filename

l'option -E de la commande grep permet de comprendre les expressions regulieres etendues.

les expressions reguliers suivantes sont considerees comme etendues:

-- ? correspond au caractere precedent zero ou une fois, donc c'est un caractere optionnel
-- + correspond au caractere precedent repeter une ou plusieurs fois
-- | alternance ou comme une logique ou un operateur

le proccessus de demarrage 
le processus de demarrage se deroule en quatre etapes principales dont certaines sont modifiees par les administrateurs,
tandis que pour les autres il suffit d'etre au courant des actions en cours.

1--etape de firmware
2-- bootloader(GRUB)
3--Stade du noyau
4--Etape d'initialisation
//
1--Bios ou encore etapes du micrologiciel

l'etapes du micrologiciel est la premiere etapes qui a lieu apres la mise sous tension de l'ordinateur. a ce stade, l'ordinateur 
est sous tension mais a besoin de commencer a exercuter un logiciel qui finira par demarrer un noyau complet.
BIOS (UEFI)= Basic input /output system ,systeme elementaire d'entree/sortie. c'est un ensemble de fonctions contenu dans la memoire
morte (ROM) de la carte mere d'un ordinateur lui permettant d'effectuer des operations elementaires lors de sa mise sous tension.
lorsque le bios est lancer , la premiere des choses qu'il fait est d'exercuter des operations de verification de l'integrite du systeme.
pour ce faire il doit lancer ce que on appelle le  POST le role du poste est d'execute l'ensemble des composant interne(CPU,RAM,Disque dur...). maintenant si le post verifie et que 
l'integrite du systeme est ok, on peut pas maintenant a chercher,charge et exercute le programme d'amorcage



--la tache finale du bios est de trouver le lecteur de demarrage approprie 
a partir des peripheriques de stockage disponibles et charger le disque de demarrage
principale (MBR) a partir de ce peripherique 


le disque d'initialisation principale est le premier secteur (ou 512bytes) du disque
il contient une table de partition et une tres petite quantite de code executable
appelle la premiere etape plus riche en fonctionnalites(GRUB ou LILO) 

2--etape du chargeur de demarrage
le role du MBR de allez chercher le chargeur d'amorcage(bootloader)
le bootloader effectuera plusieurs operations, mais la tache principale est de charger le noyau linux dans 
la memoire RAM et de l'exercuter.
une fois que cela s'est produit le noyau prend le relais pour demarrer le systeme


le role du bootloader est d'aller cherche ou se trouve le noyau (KERNEL) sur le disque dur pour le charger
en memoire vive c'est a dire la RAM ( 
la raison pour laquelle il vas chercher pour l'exercuter plustot en memoire vive est que 
la memoire ram est plus rapide que la memoire ROM (disque dur)
)

le bootloaders les plus utilises sur les machines sont le linux loader(LILO) et le 
Grand Unified Bootloader(GRUB)

CC BY («Attribution»)
La licence gratuite qui permet à quiconque d'éditer et de distribuer l'œuvre tant qu'il nomme l'auteur.

CC BY-SA («Attribution-ShareAlike»)
En tant que CC BY, sauf que l'œuvre modifiée ne peut être distribuée que sous la même licence. Le principe

CC BY-ND («Attribution-NoDerivatives»)
Comme CC BY, sauf que l'œuvre ne peut être transmise que sans modification.

CC BY-NC («Attribution-NonCommercial»)
L'œuvre peut être éditée et distribuée en nommant l'auteur, mais uniquement dans des conditions non commerciales.

CC BY-NC-SA («Attribution-NonCommercial-ShareAlike»)
Comme BY-NC, sauf que l'œuvre ne peut être partagée que dans les mêmes conditions (c'est-à-dire une licence de type copyleft).

CC BY-NC-ND («Attribution-NonCommercial-NoDerivatives»)
La licence la plus restrictive: la distribution est autorisée avec attribution de l'auteur,
mais uniquement inchangée et dans des conditions non commerciales.

informatique nécessite des tensions spécifiques avec des tolérances relativement serrées.

Lors de la première mise sous tension, un matériel spécifique à la carte mère doit être configuré 
et initialisé avant que le système puisse fonctionner. Les cartes mères utilisent une programmation
stockée dans une mémoire non volatile connue sous le nom de micrologiciel
pour gérer le matériel spécifique à la carte mère. La forme originale du micrologiciel de la carte mère
était connue sous le nom de BIOS (Basic Input / Output System).

Intel a défini une norme pour le micrologiciel avancé connue sous le nom d'EFI (Extensible Firmware Interface). 

i386: Fait référence au jeu d'instructions 32 bits associé à l'Intel 80386.

x86: Fait généralement référence aux jeux d'instructions 32 bits associés aux successeurs du 80386 tels que 80486, 80586 et Pentium.

x64 / x86-64 : Fait référence aux processeurs qui prennent en charge les instructions 32 bits et 64 bits de la famille x86.

AMD: Une référence au support x86 par les processeurs AMD.

AMD64: Une référence au support x64 par les processeurs AMD.

BRAS: Fait référence à un processeur RISC ( Reduced Instruction Set Computer ) qui n'est pas basé sur le jeu d'instructions x86. 
Couramment utilisé par les appareils embarqués, mobiles, tablettes et piles. Une version de Linux pour ARM est utilisée par le Raspberry Pi.

-- assure le controle des processeurs et des ressources de la machine physique hote
-- alloue a chaque machine virtuelle les ressources dont elle a besoin

deux types d'hyperviseurs:

-- hyperviseur de type 1:bare metal
  -s'exercuter directement le hardware
  -outil de controle de l'OS hote.
  -le modèle d'hyperviseur le plus repandu en entreprise.
  -exemple: XEN,Oracle VM, VMware ESX.

-- hyperviseur de type 2: host metal
  -s'execute a l'interieur d'un OS hote
  - cet hyperviseur s'execute en troisieme niveau au dessus du materiel
  -c'est le modele prefere souvent pqr les utilisateurs de machine Lap Top ou Desktop
  Exemple:Vmware WS, Vmware fusion, Virtual BOX.
  
  -----La Virtualisation complete.
  -permet de faire fonctionner n'importe quel systeme d'exploitation en tant qu'invité dans la machine virtuelle.
  -l'OS invité n'a pas conscience d'etre virtualisé.
  -l'OS utilise directement les composant virtuels de la VM
  -c'est la VM qui traduit les appels aux composants virtuels vers les composants physiques.
  - plus simple a realiser.
  
--la paravirtualisation: ici les applications installer sur le systeme d'exploitation utilise directement tous les resources 
materiel de la machine reel.

  differents environnement (Dev,Test,Prod).
  - utilise le kernel de l'OS Hote.
  - A son propre espace de processus et sa propre interface réseau.
  - isolé de l'hote,mais exécutée directement dessus.
  - permet de décomposer l'infracstructure applicative en petits éléments légers facile à déplooyer et  à 
   reutiliser

- permet de modulariser facilement l'application

--principe de fonctionenement de docker.
 - le developpeur crée un fichier Dockerfile contenant les commandes que docker
 va exécuter pour construire un image docker de cette application.
 
 $ docker build
 
 - l'image docker contient tout ce dont l'applicatiob a besoin pour s'ex&cuter
 correctement.
 les images Docker peuvent etre publiées dans un registre publique(Docker hub) ou privé.
 
 $ docker push image_name
 
 pour téléchager une image docker d'une application dans un host docker, il suffit d'utiliser:
 
 $ docker pull image_name
 
 la création et l'exercution d'un conteneur d'une applicatiob se fait par instanciation et exercution
 de l'image en utilisant:
 
 $  docker run image_name
 
 Avc docker run, si l'images n'existe pas dont le host, elle va procéder au téléchargement celle-ci  avant d'en créer et exécuter
 un conteneur docker.
 Docker Engine qui permet de créer le host Docker sur une machine linux(Docker daemon)
 Un client Docker qui est connecté à Docker Engine via différents connecteurs exposés par dockerd(socket,REST API,etc...)
 
 
 comparaison entre une image et un container.
 
 --Une image docker est juste un fichier package representant la template des containeurs.
   elle définit la structure du conteneur en englobe l'application containérisée et l'ensemble de 
   ses dépendances.
 -- Un conteneur représente une instance d'une image. Un conteneur est exécutée par le Docker Host.
    ce qui implique l'exécution de l'application qu'il transporte dans un environnement isolé fourni par le conteneur.

        // cours sur les design pattern
il existe 3 categories de design pattern a savoir :
--creation
--structure
--comportement

-----------------------creation
--description de la maniere dont un objet ou un ensemble d'objets peuvent
etre crees, initialises et configures.
--isolation du code relatif a la creation, a l'initialisation
 afin de rendre l'application independante de ces aspects
 exmple: abstract factory, builder,prototype, Singleton
 
----------------------structure 
descritpion de la maniere dont doivent etre connectes des objets de l'application
afin de rendre ces connections independantes des eolutions futures de l'application
exemple: adapter objet, composite, bridge,decorator, factory, proxy.

-----------------------comportement 
--description de comportements d'interaction entre objets
--gestion des interactions dynamiques entre des classes et des objets
exemples: strategy,observer,iterator,mediator,visitor,state

dans la classification GoF on compte 23 design pattern.
 nb: design pattern est un model qui permet de decrire un probleme 
 frequament recontrer et l'architechture de la solution de facon s ce que cela
 soit reutilisable frequament

Questions d'entretien
Partie 1: Questions générales
1 . JEE : C'est l’ensemble de spécifications (JPA, JAX-WS, JMS...) pour créer des applications web Java, et c'est le serveur d’application qui fournit les implémentations.

2 . La version courante de la specification JEE est : Jakarta EE 9 (le 25 Mai 2021)

3 . Les versions LTS Java sont : Java 8, 11, 17

4 . AOP : La séparation des aspects métiers et des aspects techniques d'une application est rendu possible grâce à la programmation Orienté Aspect.

5 . TDD : Test Driven Développement : développer d’abord les tests.

6 . Statefull et Statless : Deux approches pour sécuriser application web.

Statfull : la session de l’utilisateur authentifié est gérée par le serveur (avec état)
Stateless : la session de l’utilisateur authentifié est gérée par le client.
7 . Les méthodes Agile comme Scrum ou l'eXtreme Programming (XP) partent du constat que le cycle de développement en cascade est un échec.

Des cycles de développement itératifs,
L’implication du client final tout au long du projet,
Des spécifications réduites en début de projet,
8 . Maven est un outil permettant d'automatiser la gestion de projets Java. Il offre les fonctionalités suivantes:

Gérer les dépendances
Lancer la compilation des sources
Lancer les Test Unitaires
Générer les packages (jar, war, ear)
Installer les packages dans le repository
Déployer l’application dans le serveur
Générer la documentation du projet
9 . Lombok : une librairie java pour la génération de code automatique. Getter et setter …

10 . DevTools : une librairie java vient avec spring boot, permet le rechargement automatiquement des classes Java et leur configuration côté serveur. Après les changements côté serveur, il s'est déployé dynamiquement,

Partie 2: Spring framework
11 . Spring : est un framework open source pour construire et définir l'infrastructure d'une application Java, dont il facilite le développement et les tests.

12 . Spring : est un framework de l’inversion contrôle pour créer des applications entreprises, on peut le sembler avec un conteneur web (tomcat) pour avoir un serveur application.

13 . CotextLoaderListener est un listener appelé par Tomcat au moment du démarrage de l’application. Ce listener cherchera le fichier de beans spring. ce qui permet de faire l’injection des dépendances.

14 . DispatcherServlet : Une Servlet qui joue le rôle de front contrôleur, pour traiter les requêtes de clients.

15 . Differentes Configuration de Spring container :

XML (ClassPathXmlAppContext(path xml file)
Annotations (ClassPathXmlAppContext(path xml file)
Code source Java (AnnotationConfigAppContext(ClassConfig.class)
16 . Bean scope :

Singleton ( scope par défaut )
Prototype
Request (Application web)
Session (Application web)
Global-session (Application web)
17 . C'est quoi Spring container:

Création et gestion des beans (IOC)
Injecter ces beans pour l’utiliser (DI)
18 . Les types d’injection d’un Bean Spring:

Via le constructeur (recommandé)
Sous forme attribut avec l’annotation @Autoward
19 . C’est quoi la differnec entre :

@Param : Un parametre Spring Data
@RequestParam : Un parametre Spring MVC, sous la forme ( url?variable=valeur ).
@PathVariable : Un parametre Spring MVC, sous la forme ( url/varibale ).
@PathParam : Un parametre Jersey
@WebParam : Un parametre JAX-WS
20 . Nouveautés de Spring 5 : Vient avec des fonctionnalités suivantes:

Support de java 8 / 9
Support de Hibernate 5 et Junit 5
Support JPA 2
Programmation fonctionnelle
Support de kotlen
Supprt de Http2
21 . Spring boot : un module spring, ou une super couche de spring pour créer des applications spring avec zéro configuration.

22 . Avantages spring boot :

Auto configuration
Serveur web intégré (Tomcat)
Gestion de dépendances ( starters )
Création des applications standalone (jar)
23 . @SpringBootApplication : C'est une annotation de spring boot qui combine trois autres annotations:

@Configuration: pour marquer une classe comme une classe de configuration,
@EnableAutoConfiguration: pour activer la configuration automatique de spring boot.
@ComponentScan: pour scanner les différentes classes constituant les Beans de notre application.
24 . Actuator: Une dépendance spring boot pour faire le monitoring de votre application en production.
